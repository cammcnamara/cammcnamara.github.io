The Birthday Paradox

The Birthday Paradox is a classic probability problem that asks: How many people must be in a room before there's a greater than 50% chance that at least two share the same birthday? The surprising answer is just 23.

Functional Requirements:

  • Simulates thousands of trials to estimate probability of shared birthdays.
  • Compares simulation results to theoretical calculations.

My Responsibilities:

  • Built the core simulation using random number generation.
  • Optimized for speed and scalability across large trial counts.
  • Applied unit testing and mock testing (Mockito) for RNG reliability.

What I Learned:

  • Deepened understanding of probability and real-world distributions.
  • Used simulation techniques and object-oriented design patterns.
  • Practiced unit testing and efficient algorithm design.

Valet Simulator

The Valet Simulator is an object-oriented Java project designed to model a dynamic valet parking system. Vehicles are assigned to parking spots based on their attributes using intelligent algorithms. The system takes into account spot size, features (e.g., EV charger), proximity to the entrance, and overall parking lot optimization. The simulator supports multiple input strategies, assignment algorithms, and vehicle types, offering both flexibility and realism.

Both the Factory and Strategy design patterns were used to enhance code readability, modularity, and extensibility. These patterns allow for seamless integration of new features without altering existing logic, in alignment with software engineering best practices.

Functional Requirements:

  • Supports a variety of valet assignment strategies: Normal, VIP Priority, and Space Optimization.
  • Assigns vehicle types—Normal, Big, Electric, Motorcycle—to spots that match their size and requirements.
  • Factors such as EV compatibility, VIP access, and proximity to entrance are used in the assignment process.
  • Includes interchangeable input modes using Strategy Pattern: User, Random, and File-based input.
  • Employs the Factory Pattern for vehicle creation to support future extensibility while adhering to the Open/Closed Principle.

Each simulation round outputs the current state of the parking lot to the terminal. Additionally, a GUI built using Java Swing provides a user-friendly, visual representation of the system’s state.

My Responsibilities:

  • Designed and implemented core vehicle logic, including creation and assignment mechanisms.
  • Developed ParkingSpot and ParkingLot classes to manage key attributes and support assignment rules.
  • Built the Java Swing-based GUI for real-time visual tracking of parking lot status.
  • Created the Valet class to handle vehicle arrivals, departures, and apply assignment strategies.
  • Wrote comprehensive unit tests following a Test-Driven Development (TDD) approach.

What I Learned:

  • Advanced object-oriented design through abstraction, encapsulation, and inheritance.
  • Practical application of SOLID principles in collaborative development environments.
  • Creating and iterating on unit tests using JUnit to ensure reliability.
  • Refining simulation logic through TDD for better maintainability and accuracy.
  • Developing interactive Java Swing GUIs for enhanced user interaction and debugging.
