The Birthday Paradox: Probability in Action

The Birthday Paradox is a famous probability problem that challenges our intuition. It asks: How many people must be in a room before there is a greater than 50% chance that at least two share the same birthday? While most people expect the number to be quite high, the answer is surprisingly only 23! This paradox illustrates how probability works in unexpected ways.

Functional Requirements: Users can input a group size (e.g., 10, 23, 50 people). The program simulates multiple trials (e.g., 10,000 simulations) to estimate the probability. It compares the simulated results to theoretical probability calculations to verify accuracy.

I was responsible for implementing: The core probability simulation using random number generation to assign birthdays to individuals. Data collection and analysis, ensuring the program runs multiple simulations efficiently. Code optimization, ensuring the program runs quickly even for large numbers of simulations.

Our team followed an iterative development process, testing and refining the model to improve accuracy and efficiency. What I Learned: Probability and statistics: Gained a deeper understanding of how probability distributions behave in real-world scenarios. Simulation techniques: Improved my skills in random number generation and Monte Carlo simulations. Inlcuding working with unit testing RNG as well as Mockito usage. Optimization: Ensured efficient processing even with large-scale simulations.
